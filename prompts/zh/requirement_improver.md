## 要求

1. **生成新需求**：
   - 基于需求基础文档（R_base）生成全新的需求
   - 新需求必须在需求基础文档的语义范围内
   - 新需求应该具体、可操作、语义清晰、独立

2. **扩展积极需求**：
   - 基于积极需求列表，生成扩展需求
   - 扩展需求应该补充细节、相关场景、边界条件、异常处理等
   - 扩展需求应该与原始积极需求相关，但不应简单重复

3. **避免负分需求**：
   - 确保新生成的需求与已有需求列表中的内容有明显区别，避免重复或高度相似
   - 特别避免生成与负分需求相似的内容
   - 新需求应该与已有需求列表中的内容有明显区别

4. **需求质量**：
   - 所有需求应该是：
     - 具体、可操作
     - 语义清晰、独立
     - 与需求基础文档相关
   - 尽可能多地生成新需求，确保需求多样性和全面性

## 输出格式

请勿使用 JSON。直接输出纯文本，并让每条需求独占一行，以 `REQ:` 前缀开始，格式如下：

```
REQ: 第一条新需求的文本描述
REQ: 第二条扩展需求的文本描述
```

如果某条需求需要额外补充，请放在同一行的 `REQ:` 后面继续说明，不要换行。请确保：
- 只有以 `REQ:` 开头的行才表示有效需求
- 除必要的标题或说明外，不输出表格、代码块等额外结构
- 新需求与已有需求有明显区别
